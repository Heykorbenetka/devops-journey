# Repository Conventions

Данный документ описывает стандарты работы с репозиторием `devops-journey`.

---

## 1. Правила именования веток (Branch Naming)

Используем префиксы для обозначения типа задачи:

| Префикс | Назначение | Пример |
|---------|-----------|--------|
| `feat/` | Новая функциональность | `feat/add-nginx-config` |
| `fix/` | Исправление багов или опечаток | `fix/typo-in-readme` |
| `docs/` | Работа только с документацией | `docs/update-conventions` |
| `chore/` | Рутинные задачи (обновление зависимостей, `.gitignore`) | `chore/update-gitignore` |

---

## 2. Стандарт коммитов (Conventional Commits)

**Формат сообщения:** `<type>(<scope>): <subject>`

### Типы (Types)

| Тип | Назначение | Пример |
|-----|-----------|--------|
| `feat` | Новая фича (соответствует минорной версии) | `feat(git): add branch naming guide` |
| `fix` | Исправление ошибки (соответствует патч-версии) | `fix(nginx): correct config path` |
| `docs` | Изменения в документации | `docs(readme): update setup instructions` |
| `style` | Форматирование, пропущенные точки с запятой (без изменения логики) | `style: format code` |
| `refactor` | Правка кода без изменения поведения | `refactor(script): simplify logic` |
| `perf` | Улучшение производительности | `perf(docker): optimize image size` |
| `test` | Добавление или правка тестов | `test: add unit tests` |
| `build/ci` | Настройка сборки и CI/CD пайплайнов | `ci(github): add workflow` |
| `chore` | Обновление настроек репозитория | `chore: update .gitignore` |

### Области (Scopes)

Примеры областей для DevOps: <(git), (nginx), (docs), (linux).>

## 3. Правила работы с историей
Предпочтительно использование Merge Commit для вливания фич в main, чтобы сохранить историю веток.
Для мелких правок и синхронизации с main допускается Fast-forward.
## 4. Оформление сообщений
- Заголовок: до 50 символов, инфинитив/повелительное наклонение.
- Заголовок: до 50 символов, инфинитив/повелительное наклонение.
